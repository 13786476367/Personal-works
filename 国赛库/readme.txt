这套库分为以下几个部分：
1、库文件部分
2、平衡小车入门版
3、最小系统的pcb板
4、匿名飞控上位机
5、OLED的字库文件

    写这套库的目的主要是为了国赛做准备，其次我希望能通过这套库让新来的同学能够快速入门。
通过这套库，我希望新同学首先能够加强对c语言语法的熟练程度，熟练使用结构体、结构体指针、
变参函数的使用等。。。最好能够从汇编的角度理解c程序。
    同时，掌握正确的编程规范，注意程序的可读性、移植性、执行效率。这就要求掌握正确的编程
方法。少用全局变量，拥有面向对象的思维，降低各个模块之间的耦合度。
    库文件部分考虑到不同单片机硬件的不同，所以全部初始化代码均不包括单片机的硬件初始化，
这部分需要用户自行完成并进行相应的修改，如单片机IO口的使用。
    库文件部分包含了sys文件（核心文件）、i2c协议驱动文件、mpu6050文件（陀螺仪姿态解算）、
oled驱动文件（显示屏驱动）、flash文件（外部内存扩展）、anotc_v6文件（匿名飞控通讯协议）、
keyboard文件（ch455g矩阵键盘驱动文件）。
    而平衡小车入门版例程相当于以上库文件的使用例程，在使用以上库文件时可以参考一下平衡小
车入门版的使用方法。
    平衡小车入门版例程中包含了“直立环”单环直立和“直立环+速度环”双环直立，可通过
“USING_ENCODER“宏定义来控制使用单环直立还是双环直立。当注释掉该宏定义即为单环直立，定
义该宏即为双环直立，不同的小车由于陀螺仪安装的不同可能需要调节不同的中值。
    对于新同学，应该首先了解平衡小车的直立原理，和程序运行流程，这会大大加快学习效率，这
套库可以让新同学快速上手，先调用库来实现功能，再慢慢研究库的实现方式和方法，使用之后再来
理解原理同样会加快学习效率。
    在实现了平衡小车之后，一定要弄懂这套库的实现方式和方法，切不可冒进，特别是匿名飞控的
通讯协议。
    而最小系统的pcb板兼容STM32F1和F4的LQF64系列，包含了usb-micro接口、调试接口、按键、外
扩flash、oled/tft显示屏插口、蓝牙模块接口、矩阵键盘接口。
    usb-micro接口主要是为了方便使用电脑给这块最小系统供电，其次可以进行虚拟usb与电脑的通
讯。调试接口可以方便对单片机下载程序和调试。而按键可以做一些特别的功能，比如功能的切换。
    外扩flash主要是为显示屏提供中文字库、图片还有程序的一些参数。
    oled/tft显示屏插口由于留了6个IO接口可以很方便的进行oled或者的tft显示屏的切换。
    蓝牙模块可以方便单片机将数据传输到电脑的上位机，方便程序调试和观察。
    矩阵键盘的接口使用I2C通讯，节约IO口，可以用该接口连接矩阵键盘，与oled显示屏配合从而
实现菜单等功能。
    我在设计这块板子时考虑了各个比赛时基础功能的相似性，于是将这个基础功能集中到这块最小
系统板上，为以后的比赛大大提供了方便。而且矩阵键盘驱动和显示屏的驱动均考虑了和c语言标准库
的一致性，这可以使得在写程序菜单的时候可以与以前写过的c语言课程设计相似的实现方法。
   同时我还设想，由于这块最小系统拥有flash、键盘、显示屏，那么我们可以在程序中移植一个游
戏，flash可以存储游戏的图片资源、键盘可以控制游戏、显示屏显示游戏界面、板载按键可以进行游
戏和比赛功能之间的切换。这样既可以在比赛中放松自己，又不妨碍比赛的进行。
   但是在最小系统板载接口的排版时由于考虑不周，显示屏会挡住板载按键，导致按键的不太方便，
而且显示屏又与蓝牙模块接口有一定冲突，导致蓝牙模块的插入与拔出的不便。由于我已临近毕业，
所以这块最小系统的改进工作需要由你们来完成，期待改进之后的效果。
